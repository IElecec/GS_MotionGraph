# GS_MotionGraph

Minimal motion-graph pipeline for Gaussian sequence data.

## Overview

This repo currently contains a small end-to-end command-line pipeline:

1. Generate sample Gaussian `.ply` sequences under `data/char1`
2. Precompute pairwise similarity matrices
3. Build a motion graph from the saved similarity matrices

## Data Layout

The motion database is organized as:

```text
data/
  char1/
    walk/
      anim_1/
        frame_0.ply
        frame_1.ply
        ...
    attack_1/
      anim_1/
      anim_2/
    attack_2/
      anim_1/
```

Each animation directory contains frame-wise Gaussian `.ply` files named as `point_cloud_<index>.ply`.

## Requirements

The scripts assume the Python environment can import the dependencies already used in this repo, especially:

- `torch`
- `plyfile`
- `tqdm`
- `flask`

## Quick Start

### 1. Compute similarity matrices

```bash
python compute_similarity.py -m data/jyf -o outputs/jyf --window 10 --min-gap 60 --sh-degree 3
```

This writes files like:

```text
outputs/
  similarity_matrices/
    walk/
      anim_1/
        attack_1/
          anim_1/
            similarity.pt
```

Each `similarity.pt` contains:

- `distance_matrix`
- `angle_matrix`
- `window_size`
- `min_gap`
- matrix index metadata used by the motion-graph builder

### 2. Build the motion graph

```bash
python build_motion_graph.py -m data/char1 -s outputs -o outputs/motion_graph.json --distance-threshold 0.5 --top-k 5
```

If you also want one extra JSON for shortest path data:

```bash
python build_motion_graph.py -m data/jyf -s outputs/jyf -o outputs/jyf/motion_graph.json --shortest-path --distance-threshold 0.5 --top-k 3
```

Useful options:

- `--distance-threshold`: only keep transitions below this distance
- `--top-k`: keep at most top-k local-minimum transitions for each animation pair
- `--keep-dead-ends`: disable pruning to the largest strongly connected component
- `--shortest-path`: also export shortest path data

Visualization is started separately with `visualize_motion_graph.py` after `motion_graph.json` is generated.

The output is a JSON file containing:

- `nodes`: only frame references that participate in at least one transition
- `edges`: compressed sequence edges between consecutive transition nodes, plus transition edges
- `transitions`: extracted transition candidates

If `--shortest-path` is set, the builder also writes `shortest_path.json` by default. Its path cost is:

- `sum(edge.length + edge.distance)`
- for `sequence` edges, `distance` is `0`, so they only contribute frame count
- for `transition` edges, both frame count and transition distance contribute

The shortest-path JSON stores, for every source node:

- one entry for each other `action` (the node's own `action` is omitted)
- whether that target action is reachable
- the chosen target node under that action
- `total_cost`, `total_frames`, `total_transition_distance`
- the ordered node list and edge list along the path

For each transition:

- `source.frame` is the source window start frame
- `target.frame` is the target window end frame

This step only finds transition relationships. It does not synthesize any new Gaussian point clouds.

### 2.5. Visualize an existing motion graph

If you already have `motion_graph.json`, you can launch the visualizer separately:

```bash
python visualize_motion_graph.py -g outputs/jyf/motion_graph.json --mode graph --port 8765
```

Useful options:

- `--mode graph`: graph traversal only
- `--mode image`: graph traversal plus rendered image preview
- `--host`, `--port`: host / port for the Flask server
- `--frame-spacing`: horizontal spacing between frame nodes
- `--lane-spacing`: vertical spacing between animation lanes
- `--max-transition-edges`: limit the number of rendered cross-action transition edges
- `--fps`: logical playback fps used by the random walker
- `--image-manifest`: manifest generated by `render_image_library.py`; used in `--mode image`

Then open `http://127.0.0.1:8765` in your browser, or the host / port you selected.

For backward compatibility, the old `-o outputs/.../motion_graph.html` argument is accepted but ignored.

If `shortest_path.json` exists next to `motion_graph.json`, the page will also show `To <action>` buttons. Clicking one temporarily interrupts the random walker, follows the saved shortest path to that action, and then resumes random walking.

The page also includes a `Stay Within Current Action` button. When enabled, the walker only uses edges that stay inside the current action; pressing a `To <action>` button can still temporarily cross actions, and after arrival the lock continues inside the new action.

The random walker uses logical graph time instead of SVG pixel distance:

- traversal time of one edge = `edge.length / fps`
- for example, if `fps = 30` and `edge.length = 9`, the walker spends `0.3s` on that edge

The visualization is rendered as one large SVG:

- transition nodes from the same `action` are grouped into one cluster
- each cluster keeps its internal sequence edges and same-action bridge transitions
- cross-action transitions are drawn directly between clusters in the same figure
- a small animated ball performs a random walk over the visible graph edges
- the web page includes an FPS slider / numeric input so you can change walker speed interactively
- if an image manifest is available, the page also shows a preview panel that plays the corresponding rendered images as the walker moves

### 3. Fill transition frames

After `motion_graph.json` is generated, you can synthesize in-between Gaussian point clouds for its transitions:

```bash
python fill_transition_frames.py -g outputs/jyf/motion_graph.json -o outputs/jyf/transitions --sh-degree 3
```

Useful options:

- `--database`: override the database directory stored in `motion_graph.json`
- `--max-transitions`: export only the first N transitions
- `--sh-degree`: SH degree used when loading Gaussian frames

This writes one directory per transition, for example:

```text
outputs/
  transitions/
    0000_walk_anim_1_0002__attack_1_anim_1_0004/
      transition.json
      frame_0.ply
      frame_1.ply
      frame_2.ply
      frame_3.ply
```

The number of synthesized in-between frames is no longer set manually here. It is derived from `motion_graph.json`:

- `transition_edge_length = window_size - 1`
- `num_transition_frames = window_size - 2`

### 4. Render an image library

Instead of rendering a replay JSON, the current rendering path renders all normal animation frames and all synthesized transition frames to a reusable image library plus a `manifest.json`.

Example:

```bash
python render_image_library.py \
  -m "data\jyf" \
  -t "outputs\jyf\transitions" \
  -o "outputs\jyf\rendered_images" \
  --camera-position "-1.0282106399536133, 1.0925928354263306, -2.051401138305664" \
  --camera-target "0.2875324487686157, 0.6285907626152039, 0.22753290832042694"
```

This writes:

```text
outputs/
  jyf/
    rendered_images/
      manifest.json
      normal/
        action1/
          anim1/
            frame_0000.png
            frame_0001.png
            ...
      transitions/
        0000_action1_anim1_0000__action2_default_0009/
          frame_0000.png
          frame_0001.png
          ...
```

Useful options:

- `--camera-position`, `--camera-target`, `--camera-up`: use a fixed camera
- `--azimuth-deg`, `--elevation-deg`, `--distance-scale`: auto-build a camera from the first normal frame
- `--background`: background color in `[0, 1]`
- `--overwrite`: re-render PNG files even if they already exist

After the image library is ready, pass the manifest into the Flask visualizer:

```bash
python visualize_motion_graph.py \
  -g "outputs\jyf\motion_graph.json" \
  --mode image \
  --port 8765 \
  --image-manifest "outputs\jyf\rendered_images\manifest.json"
```

The page will then show a frame preview that follows the same edge traversal as the walker.

### 4.5. Render a scan view

`scan view` is a separate utility that renders a multi-view preview from a single Gaussian frame:

```bash
python render_scan_view.py --frame-path data\jyf\action1\anim1\point_cloud_9.ply --output-dir outputs\jyf\scan_view
```

Useful options:

- `--scan-start-azimuth`, `--scan-end-azimuth`, `--scan-step-azimuth`: control the azimuth sweep
- `--scan-elevation-deg`: elevation angle used by the orbit camera
- `--scan-distance-scale`: orbit camera distance multiplier
- `--background`: background color in `[0, 1]`

## Notes

- `generate_sample_char1.py` creates synthetic demo data intended for pipeline testing, not realistic production Gaussians.
- `compute_similarity.py` currently compares fixed-length windows between animation sequences.
- `build_motion_graph.py` uses the saved similarity matrices plus the original database structure to build the graph.
- `fill_transition_frames.py` uses direct interpolation of position, opacity, scaling, SH features, and quaternion rotation to synthesize transition Gaussians.
